# TASK-007: Password Security & Brute Force Protection

- **Priority:** High
- **Assigned to:** Antigravity 🤖
- **Status:** Done ✅
- **สถานะ:** เสร็จสิ้น (Done)

## ปัญหา

ระบบเดิมใช้ `auth: true` แบบ default ของ Payload CMS ซึ่ง:
- ไม่มีการจำกัดจำนวนครั้งที่ login ผิด (brute force ได้ไม่จำกัด)
- ไม่มีการล็อค account เมื่อใส่รหัสผิดหลายครั้ง
- ไม่มีความยาวรหัสผ่านขั้นต่ำ (ใส่ "1" ก็สร้างได้)
- ไม่มีเงื่อนไขตัวอักษรใหญ่/เล็ก/ตัวเลข
- Token หมดอายุทุก 2 ชั่วโมง (ต้อง login ใหม่บ่อย)

## สิ่งที่แก้ไข

### ไฟล์: `src/collections/Users/index.ts`http://localhost:3000/

1. **Brute Force Protection**
   - `maxLoginAttempts: 5` — ล็อคหลังใส่ผิด 5 ครั้ง
   - `lockTime: 600_000` — ล็อค 10 นาที (600,000 ms)

2. **Token Expiration**
   - `tokenExpiration: 28800` — 8 ชั่วโมง (ทำงานได้ทั้งวัน ไม่ต้อง login ซ้ำ)

3. **Password Policy (beforeValidate hook)**
   - ขั้นต่ำ 8 ตัวอักษร
   - ต้องมีตัวพิมพ์ใหญ่อย่างน้อย 1 ตัว (A-Z)
   - ต้องมีตัวพิมพ์เล็กอย่างน้อย 1 ตัว (a-z)
   - ต้องมีตัวเลขอย่างน้อย 1 ตัว (0-9)
   - ตัวอักษรพิเศษ: ไม่บังคับ (เพื่อไม่ให้ยุ่งยากเกินไปสำหรับ CMS ภายในทีม)

## ก่อน vs หลัง

| ด้าน | ก่อน | หลัง |
|------|------|------|
| Login ผิดได้กี่ครั้ง | ∞ (ไม่จำกัด) | 5 ครั้ง แล้วล็อค |
| ล็อค account | ไม่มี | 10 นาที |
| รหัสผ่านขั้นต่ำ | 1 ตัว | 8 ตัว |
| ตัวพิมพ์ใหญ่ | ไม่บังคับ | บังคับ 1 ตัว |
| ตัวพิมพ์เล็ก | ไม่บังคับ | บังคับ 1 ตัว |
| ตัวเลข | ไม่บังคับ | บังคับ 1 ตัว |
| ตัวอักษรพิเศษ | ไม่บังคับ | ไม่บังคับ |
| Stay login | 2 ชม. | 8 ชม. |

## ตัวอย่าง Error Message

ถ้า user ตั้งรหัส `abc123` จะได้ error:
```
รหัสผ่านไม่ผ่านเงื่อนไข: อย่างน้อย 8 ตัวอักษร, ตัวพิมพ์ใหญ่อย่างน้อย 1 ตัว (A-Z)
```

ตัวอย่างรหัสที่ผ่าน: `MyPass12`, `Tour2026!`, `AdminWow8`

## หมายเหตุ

- เมื่อ user ถูกล็อค admin สามารถ "Force Unlock" ได้จากหน้า Users
- Token expiration 8 ชม. เหมาะกับ CMS ที่ใช้ภายในทีม ไม่ใช่ระบบ public
- Payload ยังรองรับ bcrypt hashing อัตโนมัติ + CSRF protection อยู่แล้ว
- Password policy บังคับทั้งตอน create user ใหม่ และ change password
