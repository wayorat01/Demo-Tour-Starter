### 2026-04-02 — [fix] แก้บั๊ก Next.js Hydration & ขับเคลื่อนเว็บไซต์เป็นภาษาไทยล้วน

**อัปเดตระบบภาษา และป้องกัน Server-to-Client Serialization Error:**
- **ลบภาษาอังกฤษ (EN) ออกจากระบบ:** ใน `localization.config.ts` ปรับเหลือรันแค่ `locales: ['th']` และถอด fallback logic การดึงค่า `title_en` ออกจากหน้าค้นหา `SearchTour` และฝั่ง Header (`ApiCountryMenu`) ทั้งหมด เพื่อตัดปัญหาบั๊กเวลาสลับภาษาและ SEO ที่ไม่ใช้งาน
- **แก้บั๊ก Next.js Hydration Error:** ทำการเพิ่ม `JSON.parse(JSON.stringify(data))` ให้กับ Components และ Blocks ทุกตัวที่ดึง `payload.find()` หรือ `payload.findGlobal()` แล้วโยนให้ฝั่ง Client Components โดยตรง (ซึ่งจะทำให้ชิ้นส่วน ObjectId หรือ Buffered BSON object หลุดเข้าไปจน Next.js พ่น Error "Only plain objects can be passed to Client Components" และ `{th: {buffer: ...}}`)
- นำไปใช้กับโครงสร้างหลักทั้งหมด: `layout.tsx` (Global Config), `SearchOptions`, `AboutBlock` (company-info), `TestimonialBlock`, `BlogCardBlock`, `BlogListingBlock`, `GalleryListingBlock`, `TourTypeBlock`, `FestivalTourBlock` โค้ดเสถียรและทำงานได้โดยไม่ Error ในฝั่ง Client อีกต่อไป

**ไฟล์ที่เกี่ยวข้อง:**

### 2026-04-01 — [fix] จัดกึ่งกลางอัตโนมัติ (Responsive Centering) ให้ Service Card และ Tour Card ทุกรูปแบบ

**เพิ่มระบบ Auto-Center เมื่อสินค้ามีจำนวนน้อยกว่า Column ที่ตั้งไว้:**

- **Slider/Carousel Mode:** รันผ่านระบบ `CarouselContent` โดยแทรก class `justify-center` อัตโนมัติเมื่อจำนวนทัวร์หรือแพ็กเกจน้อยกว่าจำนวน Column ที่กำหนด

### 2026-04-07 — [perf] Cache ข้อมูล Header ทั้งก้อน + Optimize Block Queries

**ปัญหา:** ทุก page request ยิง DB queries 7-10 ตัวแบบ sequential แค่เพื่อแสดง Header อย่างเดียว ทำให้หน้าโหลดช้า ~1.5-2.5 วินาที

**การแก้ไข:**
- **Cache Header ทั้งก้อน:** รวมทุก query (header global + tours + flagIcons + companyInfo + citiesMap) ไว้ใน `unstable_cache` ก้อนเดียว ด้วย tag `header_full_data`
- **Cache Block-Level:** เพิ่ม `unstable_cache` ให้ TourType (`tour-groups`), Festival (`festivals`), BlogCard (`posts`)
- **Parallel Fetch:** เปลี่ยน Layout + Footer จาก sequential → `Promise.all`
- **Select Fields:** เพิ่ม `select` ใน query ทั้งหมด ดึงเฉพาะ field ที่ต้องใช้
- **ลด Limits:** ลด auto-populate tour limit จาก 1,000 → 100
- **ลด Depth:** Header global depth 2→1, Company-info depth 1→0
- **Auto-Revalidate:** เพิ่ม `revalidateTag('header_full_data')` ในทุก hook ที่เกี่ยวข้อง (header, tours, company-info)

**ผลลัพธ์:** Request ที่ 2+ (cache hit) ลดจาก ~2.3 วินาที → ~100ms

**ADR:** [ADR-003: Header Full Data Cache](../decisions/003-header-full-data-cache.md)

**ไฟล์ที่เกี่ยวข้อง:**
- `src/globals/Header/Component.tsx` — `fetchHeaderData()` + `getCachedHeaderData()`
- `src/blocks/TourType/Component.tsx`, `Festival/Component.tsx`, `BlogCard/Component.tsx` — เพิ่ม `unstable_cache`
- `src/app/(frontend)/layout.tsx` + `src/globals/Footer/Component.tsx` — `Promise.all`
- `src/utilities/getCachedCitiesMap.ts` — เพิ่ม `fetchCitiesMapDirect()`
- Hooks: `revalidateHeader.ts`, `revalidateInterTours.ts`, `revalidateCompanyInfo.ts`
